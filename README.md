# Jest Unit Tests

Проект с юнит-тестами для функций `calculateBonus` и `findTopStudent` с использованием Jest.

## Описание

Проект содержит два модуля с юнит-тестами:

### 1. calculateBonus
Функция `calculateBonus(a, b)` вычисляет бонус на основе суммы двух чисел:
- Если сумма ≤ 50, возвращает сумму
- Если сумма > 50, возвращает максимум 50

### 2. findTopStudent
Функция `findTopStudent(students)` находит студента-победителя из массива:
- Выбирает студента с максимальным количеством баллов
- Если несколько студентов имеют одинаковый максимальный балл, выбирает того, кто завершил раньше (по дате)
- Игнорирует студентов с 0 баллов
- Выводит поздравление победителю в консоль

## Покрытие кода (Code Coverage)

### Запуск измерения покрытия

Для получения отчёта о покрытии кода тестами:
```bash
npm run test:coverage
```

### Что измеряется?

Jest измеряет четыре метрики покрытия:

- **Statements (Stmts)** - процент выполненных инструкций кода
  - Каждая отдельная команда (присваивание, вызов функции, return)
  
- **Branches** - процент проверенных веток условных операторов
  - Например, для `if/else` нужно протестировать обе ветки (true и false)
  - Для тернарного оператора `? :` также две ветки
  
- **Functions (Funcs)** - процент вызванных функций
  - Каждая функция должна быть вызвана хотя бы один раз
  
- **Lines** - процент выполненных строк кода
  - Физические строки файла, которые были выполнены

### Различия между метриками

- **100% Lines**: Нужно выполнить строки 1 И 2
- **100% Branches**: Нужно протестировать случаи когда `sum > 50` (true) И когда `sum <= 50` (false)
- **100% Functions**: Нужно вызвать функцию хотя бы раз
- **100% Statements**: Все команды (if, return) должны выполниться
